# 万能应用加载器

一个轻量级的本地 Web 应用容器框架，支持多应用管理、热重载、局域网共享、多主题切换、RESTful API，以及高级标签管理、截图、版本控制、性能监控、审计日志、健康检查、个性化推荐、搜索、评分等企业级特性。将 `apps/` 目录下的每个子目录视为独立应用，提供统一入口和管理后台。

---

## 📌 与本项目母项目（星海量子通用操作系统）的关系

本组件是 **星海量子通用操作系统（StarseaOS）** 的一个独立子项目组件，采用独立于母项目的双许可证模式。

- 母项目（星海量子通用操作系统）采用 **商业专有许可证**。
- 本组件的开源社区版（AGPLv3）**不包含**母项目的任何专有代码。
- 将本组件与母项目结合使用时，需同时遵守双方的许可条款。

## 功能特性

- **应用管理**：扫描 `apps/` 目录下的所有子目录作为独立应用，每个应用只需包含 HTML 文件
- **灵活配置**：支持 `app.json` 自定义应用名称、描述、图标、分类、标签、入口文件、应用类型（MPA/SPA）及回退页面
- **管理后台**：登录保护、应用启用/禁用、批量操作、访问日志、统计仪表板、系统健康状态
- **应用收藏**：收藏常用应用，快速筛选
- **应用备注**：为每个应用添加管理备注
- **应用标签**：为应用添加多个标签，支持标签筛选、标签云、重命名、合并、批量删除
- **访问趋势**：自动统计每日访问量，以图表展示最近30天趋势
- **数据备份与恢复**：一键备份所有配置数据（密码、收藏、备注、应用配置），支持恢复
- **应用导出/导入**：将单个应用打包为 zip 导出，或导入 zip 快速部署（支持批量导入）
- **应用截图**：自动生成应用首页截图（使用 Playwright 无头浏览器），缓存 1 小时
- **版本控制**：为每个应用创建备份、列出历史版本、一键回滚到指定版本
- **性能监控**：前端上报页面加载时间、资源大小，提供统计汇总
- **审计日志**：记录管理员操作（启用/禁用、删除、导出/导入、备份/恢复等），支持查询
- **健康检查**：检测应用入口文件是否存在、是否可访问，汇总健康状态
- **推荐系统**：基于用户访问历史（标签相似度）或热门程度推荐应用
- **增强搜索**：支持按名称、描述、分类、标签、作者搜索，返回匹配度
- **评分系统**：用户可对应用进行 1~5 星评分，展示平均分与分布
- **用户认证**：支持用户注册、登录、登出，区分管理员和普通用户角色
- **热重载**：修改 HTML、CSS、JS 文件后浏览器自动刷新
- **局域网共享**：同一 Wi-Fi 下的设备均可访问
- **多主题切换**：4 种配色（暖橙、薄荷、靛蓝、玫瑰）× 明暗模式 = 8 种主题组合
- **RESTful API**：提供 `/api/v1/` 版本化 API，便于集成
- **静态资源支持**：每个应用可独立包含 CSS、JS、图片等资源
- **访问统计**：自动记录每个应用的访问次数和最后访问时间
- **安全隔离**：路径穿越防护，应用只能访问自身目录
- **免费图标**：使用 Font Awesome 免费可商用图标库
- **移动端优化**：管理后台侧边栏折叠、表格横向滚动、触摸友好的按钮大小
- **商业版订阅制**：提供 ¥99/年的专业版订阅，解锁无限应用与高级功能

1. **启动服务**：`python app.py`
2. **访问首页**：`http://127.0.0.1:8000`
3. **若未激活**：自动跳转到激活页面，或手动访问 `/api/v1/license/activate-page`
4. **输入激活码**：从官方渠道获取激活码（免费或专业版）

## 快速开始

### 环境要求

- Python 3.6 及以上版本
- pip 包管理器

### 安装依赖

```bash
pip install -r requirements.txt
# 截图功能需要额外安装 Playwright 浏览器（可选）
playwright install
```

### 启动服务

```bash
python app.py
```

默认端口为 8000，浏览器将自动打开应用列表首页。

### 管理后台

访问 http://127.0.0.1:8000/admin

默认管理员账号：`admin`，密码：`admin123`（**首次登录后请立即修改密码**）

### RESTful API

所有 API 均以 `/api/v1/` 为前缀，详细列表见下文。

## 添加新应用

在 `apps` 目录下创建子文件夹，放入 HTML 文件即可。

```bash
# 手动创建
mkdir apps/my_app
# 放入 index.html、style.css、script.js 等文件
```

### 应用配置文件

每个应用目录下可放置 `app.json` 进行配置：

```json
{
  "name": "我的应用",
  "description": "应用功能描述",
  "icon_class": "fa-gamepad",
  "category": "游戏",
  "tags": ["休闲", "HTML5"],
  "entry": "index.html",
  "type": "mpa",          // 可选 "mpa"（多页应用）或 "spa"（单页应用）
  "fallback": "index.html", // 当 type 为 spa 时，所有未匹配路径回退至此文件
  "version": "1.0",
  "author": "你的名字"
}
```

**说明**：
- `entry`：应用的入口文件，默认为 `index.html`
- `type`：应用类型，`mpa` 为传统多页应用，`spa` 为单页应用（需配合 `fallback` 处理路由）
- `fallback`：当访问不存在的静态资源（非 `.css`、`.js` 等）时，返回此文件，常用于 SPA 路由支持

## 管理后台功能详解

### 总览仪表板
- **系统健康状态**：显示 CPU、内存、磁盘使用率（需安装 `psutil`），应用总数
- **统计卡片**：应用总数、已启用数、已禁用数、HTML 文件总数、总访问次数、最近日志条数
- **访问趋势图**：展示最近30天的每日访问量折线图（使用 Chart.js）
- **数据备份与恢复**：一键备份所有配置数据（密码、收藏、备注、应用配置），支持上传备份包恢复
- **健康应用统计**：显示健康/异常应用数量（来自健康检查缓存）
- **最近审计日志**：显示最近 10 条管理员操作记录
- **应用列表**：展示所有应用的详细信息，支持批量启用/禁用/删除、单个启用/禁用、打开、删除、导出
- **备注编辑**：为每个应用添加或编辑管理备注
- **最近访问日志**：显示最近的访问记录

### 应用管理页面
- 以表格形式展示所有应用
- 显示应用名称、ID、分类、版本、类型（MPA/SPA）、状态
- 支持一键打开应用

### 访问日志页面
- 显示最近 500 条访问日志
- 日志格式：`[时间] IP - 方法 路径 - app:应用ID`

### 审计日志页面
- 显示最近 200 条管理员操作记录（启用/禁用、删除、导入/导出、备份/恢复、修改密码等）
- 包含时间、操作用户、操作类型、应用ID（如有）、详情

### 健康检查页面
- 显示所有应用的健康状态（健康/异常/禁用/错误）
- 支持一键刷新所有应用的健康状态（调用 `/api/v1/health/check/all`）
- 包含应用名称、状态、消息、最后检查时间

### 版本管理页面
- 按应用分组显示所有历史备份版本（zip 包）
- 每个应用可手动创建备份
- 每个版本可一键回滚（回滚前自动备份当前版本）

### 标签管理页面
- 显示所有标签及其使用次数（按使用次数降序）
- 支持对每个标签进行重命名、合并、删除操作
- 操作结果实时生效，并更新所有应用的 `app.json`

### 设置页面
- 显示框架配置：监听端口、应用目录、框架版本、数据目录
- 显示各配置文件路径

### 修改密码页面
- 修改管理员密码（需验证旧密码，新密码至少4位）

### 用户管理页面
- 查看所有用户（用户名、角色、创建时间）
- 修改其他用户的角色（user/admin）
- 删除其他用户

## 前端首页功能

### 搜索

- 每个应用卡片下方显示 5 星评分组件

- “探索”页根据当前登录用户的访问历史，推荐标签相似的应用

- 鼠标悬停应用卡片时，自动显示应用首页截图（由 Playwright 生成）

### 性能监控（自动）
- 在应用详情页加载时，自动上报页面加载时间和资源大小到后端
- 可在管理后台的 API 或自行扩展查看统计

### 用户认证
- 顶部显示当前用户名和角色
- 支持登录、注册、登出操作（模态框交互）

## 常见问题

### 端口被占用

```bash
python app.py --port 8080
```

### 应用无法访问

检查以下几点：
1. 应用目录下是否有 HTML 文件
2. 应用是否在管理后台中被禁用
3. 访问路径是否正确：`/app/应用ID/文件名`
4. 文件名是否大小写匹配

### 热重载不生效

确保已安装 livereload 依赖：
```bash
pip install livereload
```

### 修改默认密码

登录管理后台后，在“修改密码”页面更改，或直接编辑 `data/users.json` 文件（修改 `password_hash` 字段，需使用 `werkzeug.security.generate_password_hash` 生成）。

### 静态资源加载失败

确认资源的引用路径使用了相对路径，例如：

```html
<link rel="stylesheet" href="style.css">
<script src="script.js"></script>
<img src="images/logo.png">
```

### 访问统计不更新

确保访问的是具体文件路径（如 `/app/2/index.html`），而非目录（如 `/app/2/`）。目录访问会自动重定向到入口文件，统计会在重定向后的文件请求中更新。若仍不更新，检查 `data/apps_config.json` 文件权限。

### 系统健康状态显示 N/A

未安装 `psutil` 库，运行以下命令安装：
```bash
pip install psutil
```

安装后重启服务即可。

### 备份/恢复失败

确保 `data/` 目录有读写权限。备份包为 zip 格式，恢复时需上传同格式的 zip 文件。

### 截图功能无法使用？

请确保已安装 Playwright 并下载浏览器：

```bash
pip install playwright
playwright install
```

### 标签操作不生效？

确保应用目录下有 `app.json` 文件，标签保存在 `tags` 字段中。如果手动创建应用，请添加 `app.json`。

### 版本备份存储位置？

`data/backups/apps/` 目录下，每个应用以 `{app_id}_{timestamp}.zip` 命名。

### 批量导入的 ZIP 包有何要求？

- 可以是单层目录（直接包含 HTML）或多层目录（最外层为应用名）。
- 必须至少包含一个 `.html` 文件。
- 如果未包含 `app.json`，系统会自动创建默认配置。

### 如何开启用户注册？

默认未开放注册页面，但可通过 API `/api/v1/auth/register` 注册新用户（管理员可登录后台手动创建用户）。若需开放前台注册，可在首页登录模态框中已有“切换注册”按钮，但需用户自行点击。

### 移动端管理后台侧边栏不显示？

点击左上角汉堡按钮（三条横线）即可展开侧边栏。若仍不显示，请检查浏览器是否禁用了 JavaScript。
