# 贡献指南

感谢您对 **蓝域星河有限公司集团** 旗下开源项目的关注！我们欢迎所有形式的贡献，包括但不限于代码、文档、Bug 报告、功能建议等。


## 适用范围

本贡献指南适用于蓝域星河有限公司集团旗下所有以“星海量子”、“StarseaOS”品牌发布的开源项目。


## 行为准则

请阅读并遵守我们的 [行为准则](CODE_OF_CONDUCT.md)，共同营造友好、尊重、专业的社区氛围。


## 贡献流程

### 1. 报告 Bug 或功能建议

- 请在对应项目的 GitHub Issues 中搜索是否已存在类似问题。
- 如不存在，请创建新的 Issue，使用模板填写详细描述。

### 2. 提交代码（Pull Request）

1. **Fork 对应仓库** 到您的 GitHub 账号。
2. **创建分支**：`git checkout -b feature/your-feature-name`。
3. **编写代码**：遵循现有代码风格，添加必要的测试。
4. **签署 CLA**：在提交 Pull Request 前，请确保已阅读并同意集团的 [通用贡献者许可协议（CLA）](CONTRIBUTOR_LICENSE_AGREEMENT.md)。**提交 PR 即视为您已接受该协议**。
5. **提交 PR**：清晰描述您的修改内容、动机和测试情况。

### 3. 代码审查

- 项目维护者将审查您的 PR，可能提出修改意见。
- 请及时响应反馈，保持沟通。


## 重要提示：贡献者许可协议（CLA）

本项目采用 **双许可证** 模式（开源社区版 + 商业订阅版）。为保护项目长期发展与商业可持续性，**所有贡献者必须同意 [通用贡献者许可协议（CLA）](CONTRIBUTOR_LICENSE_AGREEMENT.md)**。

**提交 Pull Request，即视为您已阅读、理解并同意该协议**，将贡献代码的版权授予蓝域星河有限公司集团，并允许我们将贡献用于商业闭源版本。


## 开发环境搭建

```bash
# 克隆仓库
git clone https://github.com/your-username/your-project.git
cd your-project

# 安装依赖
pip install -r requirements.txt

# 启动开发服务
python app.py
```

代码风格
Python：遵循 PEP 8，使用 4 空格缩进。

前端：遵循标准 HTML5/CSS3/ES6 规范。
