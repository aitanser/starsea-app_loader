## 授权与商业版说明

本项目采用 **双许可证（AGPLv3 / 商业订阅）** 模式：

- **社区版（AGPLv3）**：免费使用，限 3 个应用，适用于个人学习、非商业项目
- **商业专业版**：¥99/年，无限应用、高级功能、闭源集成授权

开发者贡献的代码将同时用于开源社区版和商业专业版。  
提交 PR 即视为您已阅读并同意 [贡献者许可协议（CLA）](CONTRIBUTOR_LICENSE_AGREEMENT.md)。


## 贡献流程

### 1. 报告 Bug 或功能建议

- 请在对应项目的 GitHub Issues 中搜索是否已存在类似问题。
- 如不存在，请创建新的 Issue，使用模板填写详细描述。

### 2. 提交代码（Pull Request）

1. **Fork 对应仓库** 到您的 GitHub 账号。
2. **创建分支**：`git checkout -b feature/your-feature-name`。
3. **编写代码**：遵循现有代码风格，添加必要的测试。
4. **签署 CLA**：在提交 Pull Request 前，请确保已阅读并同意集团的 [通用贡献者许可协议（CLA）](CONTRIBUTOR_LICENSE_AGREEMENT.md)。**提交 PR 即视为您已接受该协议**。
5. **提交 PR**：清晰描述您的修改内容、动机和测试情况。

### 3. 代码审查

- 项目维护者将审查您的 PR，可能提出修改意见。
- 请及时响应反馈，保持沟通。


## 重要提示：贡献者许可协议（CLA）

本项目采用 **双许可证** 模式（开源社区版 + 商业订阅版）。为保护项目长期发展与商业可持续性，**所有贡献者必须同意 [通用贡献者许可协议（CLA）](CONTRIBUTOR_LICENSE_AGREEMENT.md)**。

## 提交规范

建议使用如下格式的提交信息：

```
<type>(<scope>): <subject>

<body>
```

**Type 类型**：
- `feat`：新功能
- `fix`：Bug 修复
- `docs`：文档更新
- `style`：代码格式调整（不影响功能）
- `refactor`：代码重构
- `perf`：性能优化
- `test`：测试相关
- `chore`：构建工具或辅助工具变动

**示例**：
```
feat(ui): 新增题型筛选功能

在抽屉列表中添加按题型筛选的选项，方便用户快速定位特定题型
```
